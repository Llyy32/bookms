# 07 开发与协作规范

## 7.1 分支与提交流程

- 推荐分支：`main`（稳定）+ `feature/*`（功能开发）
- 提交信息建议遵循统一前缀：`feat`、`fix`、`docs`、`refactor`
- 合并前需通过基础测试与代码评审

## 7.2 后端规范（Flask）

- 按模块组织路由、服务、数据访问层
- 业务逻辑集中在 `service` 层，避免堆在路由函数
- 统一响应结构与错误码
- 接口参数做类型与必填校验

## 7.3 前端规范（Vue3）

- 页面与组件按业务域组织目录
- API 调用统一封装，避免页面直接拼接 URL
- 表单校验规则统一管理
- 权限控制通过路由守卫与按钮级控制双保险

## 7.4 数据库规范

- 不直接删除关键业务数据，优先逻辑删除
- 对高频查询字段建立索引
- 涉及库存扣减的操作使用事务与行级锁
- 迁移脚本纳入版本管理

## 7.5 测试规范

- 后端接口需覆盖核心业务路径与异常路径
- 前端至少覆盖关键页面交互与鉴权流程
- 每次迭代需执行回归清单（借阅、归还、预约、报表、导入导出）

## 7.6 文档规范

- 需求、设计、API 文档同步维护
- 变更业务规则时先更新文档再改代码
- 新增接口必须补充请求参数与响应示例
