# 09 环境与运行部署说明

## 9.1 开发环境要求

- 操作系统：Windows / Linux / macOS
- Python：`3.11`（建议）
- Node.js：`20.x`（建议）
- MySQL：`8.0+`
- 包管理：
  - 后端：`pip + requirements.txt`
  - 前端：`npm`

## 9.2 Python 环境建议（Conda）

```bash
conda create -n bookms python=3.11 -y
conda activate bookms
```

## 9.3 数据库准备

连接参数（已在后端模板中对齐）：

- host: `localhost`
- port: `3306`
- user: `root`
- password: 以你本地 MySQL 为准（示例配置里常用 `root`）
- database: `bookms`

执行初始化 SQL（在项目根目录执行，按提示输入数据库密码）：

```bash
mysql -h localhost -P 3306 -u root -p < backend/sql/init.sql
```

### 9.3.1 演示账号（执行 `init.sql` 种子数据后）

三账号密码相同，均为：`BookMS@demo123`

- 管理员：`admin`
- 普通用户：`user001`、`user002`

若你曾导入过旧版种子（`demo_hash_*` 占位密码），请重新执行 `init.sql` 中的用户 `INSERT ... ON DUPLICATE KEY UPDATE`，以写入可用的密码哈希。

## 9.4 后端启动

```bash
cd backend
pip install -r requirements.txt
copy .env.example .env
python run.py
```

默认启动地址：`http://127.0.0.1:5000`  
健康检查：`GET /api/v1/health`

## 9.5 前端启动

```bash
cd frontend
npm install
npm run dev
```

默认访问地址：`http://127.0.0.1:5173`  
前端已配置 `/api` 代理到 `http://127.0.0.1:5000`

## 9.6 常见问题

- MySQL 连接失败：确认 MySQL 服务已启动，账户密码正确
- 端口冲突：修改 `backend/run.py` 或 `frontend/vite.config.ts` 端口
- Session 不生效：确认请求携带 Cookie，前端 `withCredentials=true`；后端已启用 `Flask-CORS` 且 `CORS_ORIGINS` 需包含前端地址（默认已含 `5173` 端口）
