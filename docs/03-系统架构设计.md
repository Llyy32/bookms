# 03 系统架构设计

## 3.1 架构概览

系统采用前后端分离架构：

- 前端：`Vue3`，负责页面渲染与交互
- 后端：`Flask API`，负责业务逻辑、鉴权、数据处理
- 数据库：`MySQL`，负责业务数据持久化

## 3.2 分层设计

建议后端按以下层次组织：

- `routes/controllers`：接口路由与参数接收
- `services`：业务规则处理（借阅上限、逾期限制等）
- `repositories/dao`：数据库读写
- `models`：数据实体映射
- `middlewares`：登录鉴权、角色校验、异常处理

## 3.3 模块划分

- 认证模块：注册、登录、退出、会话管理
- 用户模块：用户信息查询与管理
- 图书模块：图书 CRUD、库存变更
- 借阅模块：借阅、归还、逾期状态流转
- 预约模块：预约创建、取消、查询
- 报表模块：统计聚合查询
- 数据交换模块：导入导出与模板校验

## 3.4 安全设计

- 密码采用哈希算法存储（如 `bcrypt`）
- 会话通过 Cookie 传递，后端校验 Session
- 所有管理类接口进行角色校验
- 关键业务写操作记录审计日志

## 3.5 状态流转（借阅）

- 借阅记录状态建议：
  - `BORROWED`（借阅中）
  - `OVERDUE`（逾期）
  - `RETURNED`（已归还）
- 状态变更：
  - 借阅成功：`BORROWED`
  - 超过应还日期：`OVERDUE`
  - 归还：`RETURNED`

## 3.6 接口规范

- 路径前缀：`/api/v1`
- 返回结构统一：`code`、`message`、`data`
- 分页参数统一：`page`、`page_size`
- 时间字段统一采用 ISO 8601 字符串

## 3.7 日志与异常

- 统一异常处理中间件
- 区分业务异常与系统异常
- 记录请求 ID、用户 ID、接口耗时和异常堆栈
