# 01 项目概述

## 1.1 项目名称

- 中文名称：图书管理系统
- 英文名称：`bookMS`

## 1.2 项目背景

传统图书管理在图书录入、借阅登记、库存核对、统计分析等场景中，容易出现效率低、错误率高、信息不透明等问题。`bookMS` 通过前后端分离架构提供标准化的数据管理与流程管理能力，提升管理效率与用户体验。

## 1.3 建设目标

- 建立统一的图书管理与借阅管理平台
- 支持管理员与普通用户协同使用
- 提供可追踪、可统计、可导入导出的业务闭环
- 为后续 Linux 部署和系统扩展预留空间

## 1.4 角色定义

- 管理员：
  - 管理图书、用户、借阅记录
  - 处理借阅归还与预约业务
  - 查看统计报表
  - 执行导入导出
- 用户：
  - 注册、登录、查看个人信息
  - 查询图书、发起借阅与预约
  - 查看个人借阅与预约状态

## 1.5 功能边界

本期包含：

- 图书管理（增删改查、分类、库存）
- 借阅归还
- 预约管理
- 数据统计报表
- 导入导出（图书、用户、借阅记录）

本期不包含：

- 罚金计算与支付
- 多组织/多馆联动
- Linux 部署细节（后续补充）

## 1.6 约束与原则

- 认证机制采用 `Session/Cookie`
- 数据库采用 `MySQL`
- API 风格采用 RESTful 设计
- 文档先行，接口与数据结构按文档驱动开发
