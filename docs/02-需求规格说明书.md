# 02 需求规格说明书

## 2.1 范围说明

本文档定义 `bookMS` 的业务需求、功能需求与非功能需求，作为研发与测试的共同基线。

## 2.2 业务规则

- 支持用户自助注册
- 借阅规则默认值：
  - 单用户最大借阅册数：`5`
  - 借阅周期：`30` 天
  - 逾期处理：标记逾期并限制后续借阅
- 预约规则：有库存也允许预约
- 数据导入导出范围：图书、用户、借阅记录
- 报表口径：借阅排行榜、逾期统计、库存统计、用户活跃度

## 2.3 功能需求

## 2.3.1 账户与认证

- 用户可注册账号（用户名、密码、基础信息）
- 用户和管理员可登录/退出
- 登录后建立服务器端会话，浏览器通过 Cookie 维持登录状态

## 2.3.2 图书管理

- 图书新增：支持 ISBN、书名、作者、分类、出版社、总库存等字段
- 图书查询：支持关键字查询与分页
- 图书编辑：支持基础信息、库存调整
- 图书下架/删除：逻辑删除优先，保留历史借阅关联完整性

## 2.3.3 借阅归还

- 用户可发起借阅申请
- 系统校验库存与用户借阅上限
- 借阅成功后生成借阅记录并扣减可借库存
- 用户归还后更新记录状态并回补库存
- 到期未归还自动标记逾期

## 2.3.4 预约管理

- 用户可对图书发起预约（含有库存场景）
- 支持取消预约
- 管理员可查看预约队列与预约状态

## 2.3.5 报表统计

- 借阅排行榜：按图书借阅次数统计
- 逾期统计：按时间范围统计逾期数量和比例
- 库存统计：总库存、可借库存、低库存告警
- 用户活跃度：按借阅次数、预约次数排序

## 2.3.6 导入导出

- 导入对象：图书、用户、借阅记录
- 导出对象：图书、用户、借阅记录
- 支持模板校验与错误明细回传

## 2.4 非功能需求

- 可用性：核心 API 平均响应时间目标 < 500ms（常规查询）
- 安全性：密码加密存储；会话安全控制；关键操作鉴权
- 可维护性：分层设计、统一错误码、统一日志规范
- 可扩展性：预留部署与模块化扩展能力

## 2.5 验收标准（摘要）

- 功能验收覆盖已定义模块
- 借阅规则校验准确（上限、周期、逾期限制）
- 预约逻辑可用（含有库存预约）
- 四类报表可查询并返回正确统计结果
- 导入导出数据字段与模板一致
