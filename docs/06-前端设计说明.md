# 06 前端设计说明

## 6.1 技术方案

- 框架：`Vue3`
- 通信：基于 `axios/fetch` 请求 Flask API
- 会话：浏览器 Cookie 自动携带，前端关注登录态与路由守卫

## 6.2 页面结构建议

- 登录页
- 注册页
- 首页仪表盘（统计摘要）
- 图书管理页
- 借阅管理页
- 预约管理页
- 用户管理页（管理员）
- 报表页
- 导入导出页
- 个人中心页

## 6.3 路由与权限

- 游客可访问：登录、注册
- 用户可访问：图书查询、本人借阅、本人预约、个人中心
- 管理员可访问：图书管理、用户管理、报表、导入导出

## 6.4 关键交互说明

- 登录成功后跳转首页并刷新当前用户信息
- 借阅按钮根据库存、用户借阅状态动态禁用
- 预约按钮在所有库存状态下可用
- 列表页统一支持分页、搜索、筛选
- 导入支持模板下载、结果反馈、失败行提示

## 6.5 状态管理建议

- 用户登录态与角色信息全局存储
- 图书、借阅、预约数据按模块拆分状态
- 请求异常统一处理，提供标准化错误提示

## 6.6 UI 规范建议

- 统一时间格式与状态标签颜色
- 统一表格与表单组件风格
- 管理端优先保证信息密度与操作效率
