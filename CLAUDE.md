# bookMS

`bookMS` 是一个基于 `Flask API + Vue3` 的前后端分离图书管理系统。本文档用于汇总项目目标、范围与详细设计文档入口。

## 一、项目定位

- 面向中小型图书馆、阅览室或校园图书室的基础业务管理
- 支持管理员与普通用户两类角色
- 以图书管理、借阅归还、预约、统计分析、导入导出为核心功能

## 二、已确认的关键方案

- 后端框架：`Python Flask`（RESTful API）
- 前端框架：`Vue3`
- 数据库：`MySQL`
- 认证方式：`Session/Cookie`
- 系统角色：管理员、用户
- 功能范围：图书管理、借阅归还、预约、统计报表、导入导出

## 三、业务规则（当前版本）

- 允许用户自助注册
- 借阅规则默认值：
  - 最大借阅册数：`5`
  - 借阅周期：`30` 天
  - 逾期处理：仅逾期标记与借阅限制，不启用罚金
- 预约规则：有库存也可预约
- 导入导出范围：图书、用户、借阅记录
- 报表口径：借阅排行榜、逾期统计、库存统计、用户活跃度

## 六、快速启动

后端：

```bash
cd backend
pip install -r requirements.txt
copy .env.example .env
python run.py
```

前端：

```bash
cd frontend
npm install
npm run dev
```

## 八、开发规范与交付标准（强制执行）

### 8.1 分块交付流程

项目按功能块逐步推进，**严格遵循以下流程**，不得跳步：

```
开发完成一个功能块
    ↓
将该块的手工验证清单写入 docs/08-测试计划.md（格式见 8.2 节）
    ↓
执行验证清单，确认全部通过
    ↓
进入下一功能块
```

### 8.3 后端 Python 文件注释规范

**每个文件（模块级）必须有多行 docstring，包含：**

```python
"""
<文件名>模块。

在后端项目中的作用：
1. <说明该文件的核心职责>
2. <说明它与哪些模块协作>

优化建议：
- <可改进点 1>
- <可改进点 2>
"""
```

**每个函数必须有单行或多行 docstring，说明其作用、入参含义、返回值含义（如不显而易见）：**

```python
def some_func(param: str) -> Optional[User]:
    """
    <一句话说明函数作用>。

    Args:
        param: <参数含义>

    Returns:
        <返回值含义>；失败时返回 None。
    """
```

> 私有辅助函数（以 `_` 开头）可写单行 docstring。

### 8.4 前端 Vue 文件注释规范

**每个 `.vue` 文件必须在三个区块前标注结构注释：**

```vue
<!-- ===================== Template 区块 ===================== -->
<!-- 说明：本组件的视图结构，包含 <功能描述> -->
<template>
  ...
</template>

<!-- ===================== Script 区块 ===================== -->
<!-- 说明：组合式 API，包含 <数据/逻辑描述> -->
<script setup lang="ts">
  ...
</script>

<!-- ===================== Style 区块 ===================== -->
<!-- 说明：局部样式（scoped） -->
<style scoped>
  ...
</style>
```

> 组件内的关键函数/computed 也应加 `// 说明` 单行注释。

### 8.5 不确定点处理原则

遇到以下情况时，**必须停下来向用户提问，不得自行决定**：

- 业务规则存在两种以上可行方案（如分页默认值、状态机流转）
- 数据模型变更（新增字段、修改枚举值）
- 接口设计与现有文档不一致
- 前端交互设计有多种合理选择
- 任何涉及数据库 schema 的改动

**提问格式**：描述问题背景 → 列出可选方案 → 问用户选哪个。
